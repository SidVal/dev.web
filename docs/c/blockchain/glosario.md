# Glosario

- ABI (Application Binary Interface): interfaz que define cómo interactuar con contratos inteligentes en la blockchain de Ethereum, incluyendo funciones y sus parámetros.
- Approval/Allowance Mechanism: mecanismo
en ERC-20 que permite a los propietarios de
tokens autorizar a otra dirección a transferir
una cantidad específica de tokens en su
nombre.
- Array: estructura de datos que almacena una
colección de elementos (valores o variables),
cada uno identificado por uno o más índices.
- Assembly: conjunto de instrucciones de bajo
nivel que se pueden usar dentro de Solidity
para realizar operaciones más complejas o
más eficientes en términos de GAS.
- Blockchain: tecnología de registro distribuido
que permite la creación de un libro mayor
seguro y descentralizado de todas las
transacciones en una red.
- Booleano: tipo de dato que puede tener dos
valores, verdadero o falso
- Burn (Quemar): método utilizado para reducir
la cantidad total de tokens en circulación,
eliminando permanentemente tokens de la
oferta total.
- Bytes: serie de bytes de datos, útil para
manejar datos binarios o texto que no requiere
la codificación Unicode de strings.
- Constructor: función especial dentro de un
contrato que se ejecuta una sola vez al
momento de desplegar el contrato.
- dApp (Aplicación Descentralizada):
aplicación que opera en una red P2P de
computadoras en lugar de un solo servidor
centralizado.
- Debugging: proceso de identificar y corregir
errores dentro de los contratos inteligentes en
Solidity.
- DelegateCall: característica de Solidity que
permite a un contrato llamar a otro contrato en
un contexto que permite compartir el
almacenamiento entre ambos contratos.
- ERC-20 (Ethereum Request for Comment 20):
estándar técnico utilizado para la
implementación de tokens fungibles en
Ethereum.
- ERC-721 (Ethereum Request for Comment
721): estándar técnico para la implementación
de tokens no fungibles (NFTs) en Ethereum.
- Ether (ETH): criptomoneda nativa de la red
Ethereum, usada principalmente para
compensar a los mineros por realizar cálculos
y validar transacciones.
- Ethereum: plataforma de blockchain que
extiende el uso de blockchains para incluir
contratos inteligentes.
- Ethereum Node: servidor o cliente que
participa en la red Ethereum manteniendo una
copia completa de la blockchain.
- Ethereum Virtual Machine (EVM):
componente de la arquitectura de Ethereum
que actúa como un entorno de ejecución
descentralizado para contratos inteligentes.
- EIP (Ethereum Improvement Proposal):
propuestas de mejoras a Ethereum, que
pueden incluir mejoras técnicas, estándares
para tokens, o cambios en las reglas de
consenso.
- Event Listeners: funcionalidades en las dApps
que permiten reaccionar a eventos emitidos
por contratos inteligentes.
- Events (Eventos): mecanismos de los
contratos inteligentes en Solidity que facilitan
la comunicación con las interfaces de usuario
cuando se producen dentro de la blockchain.
- Fallback Function: función especial en un
contrato inteligente que se ejecuta cuando se
recibe Ether en el contrato sin ninguna otra
función especificada.
- Gas: unidad de medida que denota el costo
computacional de realizar operaciones en
Ethereum, como ejecutar transacciones o
contratos inteligentes.
- Gas Optimization: prácticas y técnicas
utilizadas para minimizar el costo de ejecución
de transacciones y contratos inteligentes en
Ethereum.
- Herencia: característica de Solidity que
permite a un contrato heredar propiedades y
funciones de uno o más contratos.
- Infura: servicio que proporciona APIs
escalables y robustas para conectar
aplicaciones con Ethereum y otras
blockchains.
- JSON RPC: protocolo ligero que permite a las
aplicaciones interactuar con una red
blockchain a través de llamadas RPC (Remote
Procedure Call) en formato JSON.
- Mapping: estructura de datos que asocia
claves con valores en un formato de tipo
diccionario.
- Metadatos: información adicional que se
puede almacenar dentro de un contrato
inteligente en Ethereum.
- Metamask: billetera digital y puente que
permite a los usuarios gestionar varias
cuentas de Ethereum e interactuar con
aplicaciones descentralizadas.
- Minting (Acuñación): proceso de creación de
nuevos tokens dentro de un contrato
inteligente.
- Modifier (Modificador): cláusula utilizada en
Solidity para cambiar el comportamiento de
las funciones, típicamente para incluir
requisitos previos a su ejecución.
- Nodos: cualquier dispositivo que contiene una
copia completa de la cadena de bloques y
mantiene la red funcionando.
- Payable: modificador de función que permite a
la función recibir Ether junto con la llamada.
- Revert: instrucción de Solidity que deshace
todas las modificaciones de estado si se
encuentra con una condición de error.
- Remix: IDE (entorno de desarrollo integrado)
basado en navegador para el desarrollo de
contratos inteligentes en Solidity.
- Smart Contract Interaction: proceso de
comunicación entre una aplicación y un
contrato inteligente en la blockchain.
- Smart Contract Upgradeability: técnicas y
patrones que permiten cambiar el
comportamiento de un contrato inteligente
después de que ha sido desplegado.
- Storage: lugar donde se almacena
permanentemente el estado del contrato en la
blockchain.
- Token Standard: conjunto de reglas definidas
que describen las funcionalidades y eventos
que un token debe implementar en Ethereum.
- Token Transfer: función para mover tokens de
una dirección a otra, cumpliendo con las reglas
del contrato inteligente.
- Transaction (Transacción): acciones
codificadas en Solidity que pueden cambiar el
estado en la blockchain.
- Transactions in Web3.js: métodos en la
biblioteca Web3.js que permiten la creación,
firma y envío de transacciones a la red
Ethereum.
- Try and Catch: estructura de manejo de
errores en Solidity que permite manejar
excepciones cuando se interactúa con otros
contratos.
- Uint: tipo de dato para números enteros sin
signo, es decir, solo positivos.
- Visibility (Visibilidad): palabras clave en
Solidity como public, private, internal, y
external que definen cómo y dónde pueden ser
accedidas las funciones o variables del
contrato.
- Wallet (Cartera): herramienta que permite a
los usuarios almacenar y gestionar sus claves
de cuenta. En Ethereum, las carteras también
permiten a los usuarios interactuar con
contratos inteligentes.
- Web3.js: biblioteca de JavaScript que permite
a las aplicaciones web interactuar con la
blockchain de Ethereum desde el navegador.
